图言 ImgSay 使用教程

欢迎使用图言 ImgSay

一个强大的 AI 图片文字识别工具，让图片中的文字轻松转化为可编辑文本

项目介绍

图言（ImgSay）是一个基于 AI 模型的图片文字识别工具，能够快速识别图片中的文字内容并将结果保存到文本文件中。

核心功能

  • 批量处理图片中的文字，提高工作效率
  • 支持多种图片格式（jpg、jpeg、png、webp、bmp）
  • 自动按顺序处理图片，确保结果的连贯性
  • 结果保存到文本文件，方便后续查看和使用
  • 支持自定义模型选择，适应不同场景需求

实现原理

图言使用了以下技术实现图片文字识别：

  • AI 模型：采用 ModelScope 平台的 Qwen3-VL-8B-Instruct 模型，该模型能够理解图片内容并提取文字
  • API 调用：通过调用 ModelScope 的 API 接口，将图片发送到云端进行处理
  • 批量处理：支持一次处理多张图片，提高识别效率
  • Base64 编码：将图片转换为 Base64 格式，方便通过 API 传输

快速开始

1. 准备工作

  • 确保您的电脑上已经下载了 ImgSay.exe 程序文件
  • 确保程序所在目录中有 config.ini 配置文件
  • 确保程序所在目录中有 图片 文件夹，用于存放需要识别的图片

2. 配置 API Key

使用图言需要一个 ModelScope API Key，获取方法如下：

  1. 访问 ModelScope 官网
  2. 注册并登录账号
  3. 完成实名认证（可选，完成后获得免费调用次数）
  4. 在个人中心获取 API Key
  5. 将获取到的 API Key 填写到 config.ini 文件中

打开 config.ini 文件，找到以下行：

API_KEY = ms-a8cb2a29-cc06-4734-a4a4-3602418dfa40

将等号后面的内容替换为您自己的 API Key。

3. 准备图片

将需要识别文字的图片放入 图片 文件夹中。支持的图片格式包括：

  • jpg
  • jpeg
  • png
  • webp
  • bmp

4. 运行程序

  1. 双击 main.exe 文件运行程序
  2. 程序会自动加载配置并开始处理图片
  3. 处理完成后，结果会保存到 识别结果.txt 文件中
  4. 按任意键退出程序

配置说明

config.ini 文件包含以下配置选项：

[ModelScope]
# 魔塔社区获取的API Key
API_KEY = ms-a8cb2a29-cc06-4734-a4a4-3602418dfa40
# 接口地址
API_URL = https://api-inference.modelscope.cn/v1/chat/completions
# 模型名称
MODEL_ID = Qwen/Qwen3-VL-8B-Instruct
# 图片文件夹路径
IMG_FOLDER = 图片/
# 识别结果保存文件
RESULT_FILE = 识别结果.txt
# 每次调用的图片数量
BATCH_SIZE = 4
  • API_KEY：ModelScope API 密钥，必须填写
  • API_URL：API 接口地址，默认值即可
  • MODEL_ID：使用的模型名称，默认值为 Qwen/Qwen3-VL-8B-Instruct，可根据需要更换为其他支持多模态的模型
  • IMG_FOLDER：图片文件夹路径，可以根据需要修改
  • RESULT_FILE：识别结果保存文件，可以根据需要修改
  • BATCH_SIZE：每次处理的图片数量，默认值为 4

模型选择说明：您可以根据需要更换不同的多模态模型，只要该模型支持图片理解和文字识别功能即可。不同模型在识别准确率、速度和支持的语言范围上可能会有所差异。

推荐模型

Qwen/Qwen3-VL-8B-Instruct

默认模型，支持多语言识别，准确率高

Doubao/Doubao-VL-7B-Instruct

字节跳动开发的多模态模型，性能优秀

GPT-4V

OpenAI 开发的先进多模态模型，识别能力强

项目开发文档

项目结构

  • main.py：主程序文件，包含核心功能实现
  • config.ini：配置文件，存储 API 密钥和其他设置
  • 图片/：存放待识别图片的文件夹
  • 识别结果.txt：保存识别结果的文件
  • main.exe：打包后的可执行文件

核心功能模块

  1. 配置加载：load_config() 函数，负责加载和验证配置文件
  2. 图片处理：get_image_base64() 函数，将图片转换为 Base64 格式
  3. 批量处理：process_batch() 函数，处理图片识别并保存结果

扩展开发

如果您想扩展项目功能，可以考虑以下方向：

功能扩展建议

  • 添加图形界面，提高用户体验
  • 支持更多图片格式和批量导入
  • 增加识别结果的格式化选项（如导出为 Word、Excel 等）
  • 添加多语言支持，适应不同语言的图片识别

模型切换指南

如果您想切换到其他支持多模态的模型，只需修改 config.ini 文件中的 MODEL_ID 配置项即可。例如：

MODEL_ID = Doubao/Doubao-VL-7B-Instruct

请注意，不同模型的 API 调用方式可能略有不同，某些模型可能需要调整请求参数。

常见问题

1. 程序运行时提示 "配置文件不存在"

请确保 config.ini 文件与 main.exe 在同一目录下。

2. 程序运行时提示 "图片文件夹不存在"

请确保 图片 文件夹与 main.exe 在同一目录下，或者修改 config.ini 中的 IMG_FOLDER 配置。

3. 识别结果为空或不准确

可能的原因：

  • API Key 无效或已过期
  • 图片质量太差，文字不清晰
  • 图片中的文字太小或太密集

4. 程序运行缓慢

处理速度取决于以下因素：

  • 图片数量和大小
  • 网络速度
  • ModelScope API 的响应速度
