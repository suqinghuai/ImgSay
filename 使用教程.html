图言 ImgSay 使用教程

欢迎使用图言 ImgSay

一个强大的 AI 图片文字识别工具，让图片中的文字轻松转化为可编辑文本

版本：2.0.0

项目介绍

图言（ImgSay）是一个强大的AI驱动的图片文字识别工具，专门设计用于高效、准确地从图片中提取文字内容。相比传统OCR软件，图言利用多模态AI模型，提供更高的识别准确率和更智能的文字理解能力。

核心优势

  • 🎯 超高准确率：采用多模态模型，识别准确率远超传统OCR
  • 🚀 批量处理：支持同时处理多张图片，大幅提升工作效率
  • 🔄 智能排序：自动按自然顺序处理图片，确保结果连贯性
  • 💾 结果导出：识别结果自动保存到文本文件，方便后续使用
  • 🛠️ 灵活配置：支持自定义AI模型、批量大小等参数
  • 🆓 免费使用：支持使用免费的API接口调用大模型识别提取文字

适用场景

  • 📚 文档数字化：将纸质文档、书籍页面快速转换为可编辑文本
  • 📊 数据提取：从图表、报表图片中提取关键数据信息
  • 📝 笔记整理：识别手写笔记、会议记录等图片内容
  • 🌐 多语言识别：支持中文、英文等多种语言文字识别
  • 🏢 办公自动化：批量处理工作中的图片文字识别需求

实现原理

图言使用了以下技术实现图片文字识别：

  • AI 模型：采用 ModelScope 平台的 Qwen3-VL-8B-Instruct 模型，该模型能够理解图片内容并提取文字
  • API 调用：通过调用 ModelScope 的 API 接口，将图片发送到云端进行处理
  • 批量处理：支持一次处理多张图片，提高识别效率
  • Base64 编码：将图片转换为 Base64 格式，方便通过 API 传输

快速开始

环境要求

  • Windows 10/11 操作系统

安装步骤

  1. 下载安装包
    • 下载最新版本的 ImgSay.exe 可执行文件
    • 确保程序目录结构完整
  2. 获取API密钥（以ModelScope平台为例）
    • 访问 ModelScope官网 注册账号
    • 完成实名认证，绑定阿里云账号（认证后获得免费调用额度）
    • 在个人中心获取API Key
  3. 配置API密钥
    # 编辑 config.ini 文件 API_KEY = your_api_key_here
  4. 准备图片
    • 将需要识别的图片放入 图片/ 文件夹
    • 支持格式：jpg、jpeg、png、webp、bmp（建议使用jpg格式，其它格式尚未经过严格测试）
  5. 运行程序
    • 双击运行 ImgSay.exe
    • 打开GUI界面后，可直接在界面里修改配置并开始识别

GUI界面使用

  1. 配置参数
    • 点击"配置"标签页
    • 填写API Key等必要参数
    • 点击"保存配置"按钮
  2. 开始识别
    • 点击"运行"标签页
    • 点击"开始识别"按钮
    • 观察进度条和日志输出
  3. 查看结果
    • 识别完成后，点击"打开识别结果文件"按钮
    • 或直接查看配置的结果文件
  4. 控制功能
    • 暂停识别：点击"暂停识别"按钮
    • 继续识别：点击"继续识别"按钮

配置说明

GUI界面配置

图言 ImgSay v2.0.0 提供了直观的GUI界面，您可以在界面中直接配置所有参数，无需手动编辑config.ini文件。

配置步骤

  1. 打开配置页面
    • 启动程序后，点击顶部的"配置"标签页
    • 所有配置选项分为三个卡片：API配置、路径配置和高级设置
  2. API配置（兼容OpenAI格式）
    配置项 说明 示例值
    API Key 多模态大模型的API密钥 your_api_key_here
    API URL API调用地址 https://api-inference.modelscope.cn/v1/chat/completions
    Model ID 使用的AI模型 Qwen/Qwen3-VL-8B-Instruct
  3. 路径配置
    配置项 说明 示例值
    图片文件夹 存放待识别图片的目录 图片/
    结果文件 保存识别结果的文件 识别结果.txt
  4. 高级设置
    配置项 说明 示例值
    批处理数量 每次处理的图片数量 4
    识别失败处理 处理失败时的操作 跳过 (可选：跳过、重试、结束)
  5. 保存配置
    • 配置完成后，点击"保存配置"按钮
    • 系统会自动将配置保存到config.ini文件中

配置文件说明

config.ini 文件包含以下配置选项：

[ModelScope]
# 魔塔社区获取的API Key
API_KEY = your_api_key_here
# 接口地址
API_URL = https://api-inference.modelscope.cn/v1/chat/completions
# 模型名称
MODEL_ID = Qwen/Qwen3-VL-8B-Instruct
# 图片文件夹路径
IMG_FOLDER = 图片/
# 识别结果保存文件
RESULT_FILE = 识别结果.txt
# 每次调用的图片数量
BATCH_SIZE = 4
# 识别失败处理策略
ERROR_HANDLING = skip
  • API_KEY：ModelScope API 密钥，必须填写
  • API_URL：API 接口地址，默认值即可
  • MODEL_ID：使用的模型名称，默认值为 Qwen/Qwen3-VL-8B-Instruct，可根据需要更换为其他支持多模态的模型
  • IMG_FOLDER：图片文件夹路径，可以根据需要修改
  • RESULT_FILE：识别结果保存文件，可以根据需要修改
  • BATCH_SIZE：每次处理的图片数量，默认值为 4
  • ERROR_HANDLING：处理失败时的操作，可选值为 skip（跳过）、retry（重试）、stop（结束）

模型选择说明：您可以根据需要更换不同的多模态模型，只要该模型支持图片理解和文字识别功能即可。不同模型在识别准确率、速度和支持的语言范围上可能会有所差异。

命令行模式

如果您需要使用命令行模式运行，可以直接编辑config.ini文件，然后使用 --cli 参数运行程序：

ImgSay.exe --cli

项目开发文档

项目结构

图言-ImgSay/ ├── main.py # 主程序文件 ├── config.ini # 配置文件 ├── ImgSay.exe # 可执行程序 ├── 图片/ # 待识别图片文件夹 ├── 识别结果.txt # 识别结果输出文件 ├── main.ico # 程序图标 ├── requirements.txt # 项目依赖文件 └── README.md # 项目说明文档

核心功能模块

  1. 配置加载：load_config() 函数，负责加载和验证配置文件
  2. 图片处理：get_image_base64() 函数，将图片转换为 Base64 格式
  3. 批量处理：process_batch() 函数，处理图片识别并保存结果
  4. GUI界面：ImgSayApp 类，提供直观的图形用户界面

技术栈

  • 前端：Tkinter（Python内置GUI库）
  • 后端：Python 3.8+
  • API：ModelScope多模态API
  • 工具库：requests, configparser, base64, threading

模型切换指南

如果您想切换到其他支持多模态的模型，只需修改 config.ini 文件中的 MODEL_ID 配置项即可。例如：

MODEL_ID = Doubao/Doubao-VL-7B-Instruct

请注意，不同模型的 API 调用方式可能略有不同，某些模型可能需要调整请求参数。

常见问题

1. 程序运行时提示 "配置文件不存在"

请确保 config.ini 文件与 main.exe 在同一目录下。

2. 程序运行时提示 "图片文件夹不存在"

请确保 图片 文件夹与 main.exe 在同一目录下，或者修改 config.ini 中的 IMG_FOLDER 配置。

3. 识别结果为空或不准确

可能的原因：

  • API Key 无效或已过期
  • 图片质量太差，文字不清晰
  • 图片中的文字太小或太密集

4. 程序运行缓慢

处理速度取决于以下因素：

  • 图片数量和大小
  • 网络速度
  • ModelScope API 的响应速度

常见问题

Q1: 如何获取免费的API调用额度？

A: 在ModelScope完成实名认证后，每天可获得2000次免费调用，单个模型理论上不超过500次。

Q2: 支持哪些图片格式？

A: 支持jpg、jpeg、png、webp、bmp等常见图片格式。

Q3: 识别结果不准确怎么办？

A: 可能原因包括：

  • API密钥无效或已过期
  • 图片质量太差，文字不清晰
  • 图片中文字太小或太密集
  • 网络连接不稳定

Q4: 可以离线使用吗？

A: 目前需要网络连接调用云端AI模型，如果需要离线使用，可以通过ollama在本地部署AI模型。

Q5: 如何切换其他AI模型？

A: 只需在config.ini中修改MODEL_ID配置项即可切换到其他支持多模态的模型。
