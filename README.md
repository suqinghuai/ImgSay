# 图言 ImgSay - AI图片文字识别工具


## 📖 项目介绍

**图言 ImgSay** 是一个强大的AI驱动的图片文字识别工具，专门设计用于高效、准确地从图片中提取文字内容。相比传统OCR软件，图言利用多模态AI模型，提供更高的识别准确率和更智能的文字理解能力。

### ✨ 核心优势

- 🎯 **超高准确率**：采用多模态模型，识别准确率远超传统OCR
- 🚀 **批量处理**：支持同时处理多张图片，大幅提升工作效率
- 🔄 **智能排序**：自动按自然顺序处理图片，确保结果连贯性
- 💾 **结果导出**：识别结果自动保存到文本文件，方便后续使用
- 🛠️ **灵活配置**：支持自定义AI模型、批量大小等参数
- 🆓 **免费使用**：支持使用免费的API接口调用大模型识别提取文字

## 🎯 适用场景

- 📚 文档数字化：将纸质文档、书籍页面快速转换为可编辑文本
- 📊 数据提取：从图表、报表图片中提取关键数据信息
- 📝 笔记整理：识别手写笔记、会议记录等图片内容
- 🌐 多语言识别：支持中文、英文等多种语言文字识别
- 🏢 办公自动化：批量处理工作中的图片文字识别需求

## 🚀 快速开始

### 环境要求

- Windows 10/11 操作系统

### 安装步骤

1. **下载安装包**
   - 下载最新版本的 ImgSay.exe 可执行文件
   - 确保程序目录结构完整

2. **获取API密钥（以ModelScope平台为例）**
   - 访问 ModelScope官网 注册账号
   - 完成实名认证，绑定阿里云账号（认证后获得免费调用额度）
   - 在个人中心获取API Key

3. **配置API密钥**
   ```ini
   # 编辑 config.ini 文件
   API_KEY = your_api_key_here
   ```

4. **准备图片**
   - 将需要识别的图片放入 图片/ 文件夹
   - 支持格式：jpg、jpeg、png、webp、bmp（建议使用jpg格式，其它格式尚未经过严格测试）

5. **运行程序**
   - 双击运行 ImgSay.exe
   - 打开GUI界面后，可直接在界面里修改配置并开始识别

### GUI界面使用

1. **配置参数**
   - 点击"配置"标签页
   - 填写API Key等必要参数
   - 点击"保存配置"按钮

2. **开始识别**
   - 点击"运行"标签页
   - 点击"开始识别"按钮
   - 观察进度条和日志输出

3. **查看结果**
   - 识别完成后，点击"打开识别结果文件"按钮
   - 或直接查看配置的结果文件

4. **控制功能**
   - 暂停识别：点击"暂停识别"按钮
   - 继续识别：点击"继续识别"按钮


## ⚙️ 配置说明

### GUI界面配置

图言 ImgSay v2.0.0 提供了直观的GUI界面，您可以在界面中直接配置所有参数，无需手动编辑config.ini文件。

#### 配置步骤

1. **打开配置页面**
   - 启动程序后，点击顶部的"配置"标签页
   - 所有配置选项分为三个卡片：API配置、路径配置和高级设置

2. **API配置（兼容OpenAI格式）**
   | 配置项 | 说明 | 示例值 |
   |--------|------|--------|
   | API Key | 多模态大模型的API密钥 | `your_api_key_here` |
   | API URL | API调用地址 | `https://api-inference.modelscope.cn/v1/chat/completions` |
   | Model ID | 使用的AI模型 | `Qwen/Qwen3-VL-8B-Instruct` |

3. **路径配置**
   | 配置项 | 说明 | 示例值 |
   |--------|------|--------|
   | 图片文件夹 | 存放待识别图片的目录 | `图片/` |
   | 结果文件 | 保存识别结果的文件 | `识别结果.txt` |

4. **高级设置**
   | 配置项 | 说明 | 示例值 |
   |--------|------|--------|
   | 批处理数量 | 每次处理的图片数量 | `4` |
   | 识别失败处理 | 处理失败时的操作 | `跳过` (可选：跳过、重试、结束) |

5. **保存配置**
   - 配置完成后，点击"保存配置"按钮
   - 系统会自动将配置保存到config.ini文件中

### 配置说明

- **API Key**：必须填写有效的ModelScope API密钥，否则无法调用AI模型
- **图片文件夹**：必须存在且包含待识别的图片
- **批处理数量**：建议设置为4-8，根据电脑性能和网络状况调整
- **识别失败处理**：
  - 跳过：遇到失败时跳过当前批次，继续处理下一批
  - 重试：遇到失败时无限重试，直到成功
  - 结束：遇到失败时停止整个处理过程

### 命令行模式配置

如果您需要使用命令行模式运行，可以直接编辑config.ini文件：

```ini
[ModelScope]
API_KEY = your_api_key_here
API_URL = https://api-inference.modelscope.cn/v1/chat/completions
MODEL_ID = Qwen/Qwen3-VL-8B-Instruct
IMG_FOLDER = 图片/
RESULT_FILE = 识别结果.txt
BATCH_SIZE = 4
ERROR_HANDLING = skip
```

然后使用 `--cli` 参数运行程序：
```bash
ImgSay.exe --cli
```

## 🔧 技术实现

### 核心功能模块

1. **配置加载模块** (load_config())
   - 读取和验证配置文件
   - 检查API密钥和图片文件夹

2. **图片处理模块** (get_image_base64())
   - 将图片转换为Base64格式
   - 支持多种图片格式

3. **批量处理模块** (process_batch())
   - 调用AI模型API进行文字识别
   - 实现重试机制和错误处理
   - 保存识别结果到文件

4. **GUI界面模块** (ImgSayApp)
   - 提供直观的图形用户界面
   - 支持实时配置管理
   - 显示处理进度和日志
   - 支持暂停/继续功能

### 技术栈

- **前端**：Tkinter（Python内置GUI库）
- **后端**：Python 3.8+
- **API**：ModelScope多模态API
- **工具库**：requests, configparser, base64, threading

## ❓ 常见问题

### Q1: 如何获取免费的API调用额度？
A: 在ModelScope完成实名认证后，每天可获得2000次免费调用，单个模型理论上不超过500次。

### Q2: 支持哪些图片格式？
A: 支持jpg、jpeg、png、webp、bmp等常见图片格式。

### Q3: 识别结果不准确怎么办？
A: 可能原因包括：
- API密钥无效或已过期
- 图片质量太差，文字不清晰
- 图片中文字太小或太密集
- 网络连接不稳定

### Q4: 可以离线使用吗？
A: 目前需要网络连接调用云端AI模型，如果需要离线使用，可以通过ollama在本地部署AI模型。

### Q5: 如何切换其他AI模型？
A: 只需在config.ini中修改MODEL_ID配置项即可切换到其他支持多模态的模型。
